# FMO 中继控制功能说明

本文档说明本分支新增的 FMO 控制相关功能，方便维护者评估和后续调整。

## 新增功能

### 通联卡片呼号跳转 QRZ

在通联详情卡片中，以下呼号会显示为可点击链接：

- 卡片标题呼号
- 接收方呼号
- 发送方呼号
- 中继管理员呼号

点击后会在新页面打开 QRZ 查询页，例如：

```text
https://www.qrz.com/db/BH1KZQ
```

### FMO 中继控制页

“更多”页面中的“远程控制”已调整为“FMO 控制”，页面上方新增 FMO 中继控制区域，原 APRS 远程控制保留在下方。

中继控制区域支持：

- 显示当前中继
- 刷新当前中继
- 切换上一中继
- 切换下一中继
- 打开中继列表并选择中继
- 手动填写控制地址

控制地址默认使用：

```text
ws://192.168.31.146
```

也会优先使用项目设置里已经选择的 FMO 地址。

### 实时仪表盘

顶部主导航新增“仪表盘”页面，用于查看当前 FMO 的实时通联概览。

仪表盘会直接请求 FMO API，不依赖本地 IndexedDB 数据。页面内容包括：

- 顶部显示当前正在使用的中继名称
- 显示控制地址和中继 UID
- 显示最近 20 条通联
- 每条通联显示呼号、时间、留言、模式、中继
- 默认每 5 秒自动刷新一次
- 中继名称支持点击切换到对应中继

该页面适合放在浏览器中常开，用作实时通联观察面板。

### 从通联日志切换中继

通联日志列表中的“中继”列现在可以点击中继名称。点击后会尝试将当前 FMO 设备切换到该中继。

通联详情卡片中的“中继”名称也支持同样的点击切换。

## 实现方式

项目中原本已经封装了 FMO WebSocket API，例如：

- `getCurrentStation`
- `getAllStations`
- `setCurrentStation`
- `nextStation`
- `prevStation`

本功能复用了这些接口。

由于目前通联日志中保存的是：

- `relayName`
- `relayAdmin`

没有保存中继 UID，因此从日志切换中继时采用以下流程：

1. 读取 FMO 当前中继列表
2. 使用日志里的 `relayName` 按名称匹配中继
3. 找到匹配项后使用该中继的 `uid` 调用 `setCurrentStation`
4. 等待约 700ms 后重新读取当前中继，更新页面状态

## 已知限制

- 如果中继名称在 FMO 中继列表中不存在，则无法自动切换。
- 如果多个中继名称完全相同，当前实现会使用第一个匹配结果。
- 日志中没有保存中继 UID，因此名称匹配是当前可用的折中方案。
- 切换中继会真实改变 FMO 设备当前连接状态，属于外部可见操作。

```text
npm run build
```

```text
192.168.31.146
```

验证内容：

- 可读取当前中继
- 可加载中继列表
- 可通过控制页切换中继
- 可通过日志中的中继名称触发切换
- 可通过仪表盘查看当前中继和最近 20 条通联
